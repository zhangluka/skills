# AI驱动项目模板生成方法论

## 🎯 核心理念

**从"工具驱动"到"文档+AI驱动"的范式转换**

传统的项目初始化依赖CLI工具，需要复杂的配置和学习成本。AI驱动模板生成通过结构化文档指导AI完成项目创建，实现更智能、更灵活的模板生成。

### 🏗️ 三层架构

```

└─────────────────────────────────────┘
```

## 🎯 核心原则

### 1️⃣ 文档驱动原则
- **单一入口**: 通过一个主prompt整合所有需求
- **模块化引用**: 专门文档处理不同方面（配置、代码、部署等）
- **标准化格式**: 统一的标记和结构规范

### 2️⃣ AI适配原则
- **模型优化**: 针对不同AI模型的特点调整prompt结构
- **明确指令**: 提供清晰的执行步骤和输出要求
- **容错设计**: 考虑AI理解差异，提供多重验证

### 3️⃣ 质量保证原则
- **多级验证**: 结构验证、功能验证、质量验证
- **可追溯性**: 每个生成结果都能追溯到具体文档
- **持续改进**: 基于反馈不断优化模板和指令

## 🔄 标准化流程

### 阶段1: 需求分析 📊
```
需求收集 → 技术选型 → 约束定义 → 目标设定
    ↓
输出: 项目需求清单
```

**关键活动:**
- 确定技术栈和版本要求
- 识别特殊配置和魔改需求
- 定义部署和环境要求
- 收集质量标准和验证要求

### 阶段2: 文档设计 📝
```
需求分解 → 模块规划 → 模板设计 → 标记定义
    ↓
输出: 结构化文档体系
```

**关键活动:**
- 设计主prompt结构
- 规划引用文档模块
- 创建代码模板标记
- 定义变量替换规则

### 阶段3: 内容创作 ✍️
```
指令编写 → 模板创建 → 示例编写 → 验证设计
    ↓
输出: 完整的AI指令文档
```

**关键活动:**
- 编写详细的AI指令
- 创建可复用的代码模板
- 提供实际使用示例
- 设计验证检查清单

### 阶段4: 测试验证 🧪
```
AI执行 → 结果检查 → 问题修复 → 优化迭代
    ↓
输出: 可用的模板生成技能
```

**关键活动:**
- 使用AI生成测试项目
- 验证生成结果的完整性
- 修复发现的问题和缺陷
- 基于反馈进行优化

### 🎯 文档标准化

#### 主Prompt模板
```markdown
# [Framework]项目模板生成AI指令

## 🎯 任务目标
[明确的任务描述和目标]

## 📋 执行步骤
1. [步骤1 - 理解需求]
2. [步骤2 - 生成基础]
3. [步骤3 - 应用配置]
4. [步骤4 - 验证完整性]

## 📝 参考文档
1. 01-overview.md - 项目概述和技术栈
2. 02-config.md - 配置和魔改要求
3. 03-structure.md - 目录结构
4. 04-templates.md - 代码模板
5. 05-deployment.md - 部署配置
6. 06-usage.md - 使用说明

## 🔍 输出要求
[具体的输出格式和验证要求]
```

#### 代码模板标记规范
```markdown
### [CODE_TEMPLATE] 文件名
```language
// 可直接复制的代码
```

### 变量替换
${VARIABLE_NAME} // 占位符
```

## 🛠️ AI指令优化策略

### 🎯 Prompt结构优化

#### 1️⃣ 层次化组织
```markdown
# 顶层目标
## 中层步骤
### 底层细节
```

#### 2️⃣ 明确指令
- 使用命令式语言 ("请生成", "确保", "验证")
- 避免歧义表达
- 提供具体示例

#### 3️⃣ 输出规范
- 定义清晰的输出格式
- 指定验证检查项
- 要求错误处理

### 🔄 模型适配

#### DeepSeek V3 特点
- 理解中文指令能力强
- 代码生成质量高
- 需要明确的结构指导

#### 适配策略
- 使用中文编写主要指令
- 提供详细的中文注释
- 保持清晰的文档结构
