# AI Prompt优化指南

## 🎯 概述

本指南提供针对不同AI模型的Prompt优化策略，帮助提高项目模板生成的质量和成功率。

## 🤖 AI模型特性分析

### DeepSeek V3
#### 优势
- **中文理解能力强**: 特别擅长处理中文指令
- **代码生成质量高**: 生成的代码语法正确，结构清晰
- **结构化理解**: 能够很好地理解层次化的文档结构

#### 优化策略
```markdown
# 使用中文主要指令
- 主要描述和指令使用中文
- 代码注释提供中文说明
- 保持文档结构清晰简洁

# 明确的分段指导
- 使用数字列表明确步骤
- 每个步骤提供具体行动指导
- 避免过于复杂的嵌套结构
```

### GPT-4
#### 优势
- **英文理解精确**: 英文指令理解更准确
- **上下文理解**: 能够很好地理解长上下文
- **推理能力强**: 能够理解复杂的逻辑关系

#### 优化策略
```markdown
# 详细的上下文描述
- 提供充分的背景信息
- 明确各部分之间的关系
- 给出具体的使用场景

# 逻辑性强的结构
- 明确的逻辑递进关系
- 详细的技术要求说明
- 完整的验证标准
```

### Claude-3
#### 优势
- **代码质量高**: 生成的代码质量优秀
- **安全性好**: 注意代码安全和最佳实践
- **文档理解**: 能够很好地理解文档型指令

#### 优化策略
```markdown
# 强调最佳实践
- 明确安全和质量要求
- 提供行业标准和规范
- 强调代码可维护性

# 文档化风格
- 使用文档化的表达方式
- 提供详细的使用说明
- 包含完整的示例
```

### 🎯 优化要点

#### 1️⃣ 角色定义优化
```markdown
# ❌ 不好的定义
你是一个前端开发者

# ✅ 好的定义
你是一个专业的前端项目生成专家，精通现代前端技术栈，具有丰富的大型项目经验，能够生成企业级的、生产就绪的项目模板
```

#### 2️⃣ 步骤描述优化
```markdown
# ❌ 不好的描述
1. 生成项目
2. 配置文件
3. 创建结构

# ✅ 好的描述
1. **理解项目需求**：仔细阅读所有规范文档，理解技术栈、项目结构、特殊配置要求，确保准确把握所有需求细节
2. **生成项目基础**：创建标准的[Framework]项目结构，集成必要的构建工具和开发环境，确保基础架构稳固
3. **应用自定义配置**：重点参考`02-[config]-config.md`，严格应用所有自定义配置，不得遗漏任何魔改要求
```

#### 3️⃣ 要求明确化
```markdown
# ❌ 不明确的要求
- 使用TypeScript
- 配置构建工具
- 添加测试

# ✅ 明确的要求
- ✅ TypeScript 5.0+，开启strict模式
- ✅ Vite 4.0+作为构建工具，配置路径别名和优化选项
- ✅ Jest + Testing Library，覆盖率要求>80%
```

## 🔄 A/B测试策略

### 🧪 测试方法
```bash

prompt-v1.md  # 原始版本
prompt-v2.md  # 优化版本

# 使用相同的AI模型进行测试
for i in {1..10}; do
    echo "测试 v1 - 第${i}次"
    ai-generate --prompt prompt-v1.md --output result-v1-${i}.md
    
    echo "测试 v2 - 第${i}次"  
    ai-generate --prompt prompt-v2.md --output result-v2-${i}.md
done

# 评估结果
node tools/evaluate-prompts.js result-v1-* result-v2-*
```

- **可用性**: 生成项目的可用性

## 🎯 最佳实践总结

### ✅ 推荐做法
- **明确的角色定义**: 给AI清晰的专家角色
- **结构化的指令**: 使用层次化的文档结构
- **具体的要求**: 避免模糊的描述
- **验证机制**: 提供明确的验证标准
- **格式化输出**: 要求结构化的输出格式

### ❌ 避免做法
- **模糊的指令**: 避免不够具体的描述
- **复杂的嵌套**: 避免过于复杂的文档结构
- **缺少验证**: 避免没有验证标准
- **不一致的格式**: 避免格式不统一
