# 创建新模板指南

## 🎯 概述

本指南将帮助您为AI驱动模板生成系统创建新的项目模板，扩展系统的技术栈支持。

## 📋 前置条件

- 熟悉目标技术栈
- 了解AI Prompt编写
- 具备项目配置经验
- 掌握Markdown文档编写

## 🏗️ 模板结构标准

每个技术栈模板需要遵循以下目录结构：

```
[framework]/
├── 00-main-prompt.md         # 主指令文档 (必需)
├── 01-project-overview.md    # 项目概述和技术栈 (必需)
├── 02-[config-tool]-config.md # 配置工具魔改说明 (必需)
├── 03-structure-templates.md # 目录结构模板 (必需)
├── 04-code-templates.md      # 代码片段模板 (必需)
├── 05-deployment-templates.md # 部署相关配置 (必需)
└── 06-usage-instructions.md  # 使用说明和验证 (必需)
```

## 📝 文档编写规范

### 1. 主指令文档 (00-main-prompt.md)

#### 基本结构
```markdown
# [Framework]项目模板生成AI指令

## 🎯 任务目标
[明确的任务描述，包含AI角色定义]

## 📋 执行步骤
1. [步骤1 - 理解需求]
2. [步骤2 - 生成基础]
3. [步骤3 - 应用配置]
4. [步骤4 - 生成结构]
5. [步骤5 - 填充代码]
6. [步骤6 - 验证完整性]

## 🎯 核心要求
### 必须包含的特性
- ✅ [特性1]
- ✅ [特性2]
- ...

### 关键配置要求
- [配置要求1]
- [配置要求2]
- ...

## 📝 参考文档
[列出所有参考文档]

## ⚠️ 重要提醒
1. [提醒1]
2. [提醒2]
- ...

## 🔍 输出格式要求
[定义期望的输出格式]

## 🚀 开始执行
[启动指令]
```

#### 编写要点
- **目标明确**: 清晰定义AI角色和任务
- **步骤具体**: 每个步骤都可执行和验证
- **要求详细**: 明确必须包含的功能和配置
- **格式标准**: 统一的输出格式要求

## 🎯 AI指令优化

### 针对DeepSeek V3的优化
- 使用中文编写主要指令
- 提供详细的中文注释
- 保持清晰的文档结构
- 避免过于复杂的嵌套

### 通用优化原则
- **明确性**: 使用命令式语言
- **结构性**: 层次化组织内容
- **完整性**: 覆盖所有必要方面
- **可验证**: 提供检查标准

## 🧪 模板验证

### 自动化验证
```bash
# 结构验证
node tools/template-validator.js --template [framework]

# 内容验证
node tools/content-checker.js --template [framework]
```

### 手动验证
1. **AI生成测试**: 使用AI生成测试项目
2. **功能验证**: 验证生成的项目可正常运行
3. **完整性检查**: 确保所有文件和配置完整
4. **文档准确性**: 验证文档内容正确性
