# 谷雨计划 · 个人素材库

> 核心理念：把你们自己的历史内容变成可搜索的素材库。
> 写文章时，从真实经历里找素材，比从零编造更有说服力。

## 何时使用

- 写文章时需要引用自己的真实经历
- 需要找之前测试过的工具/场景的记录
- 文章缺少"第一手体验"的支撑
- 需要找一个具体案例来开头或论证
- grain-proofreading 发现"缺少细节"，需要从素材库补充

## 素材库结构

```
_materials/
├── README.md                    # 素材库索引和使用说明
├── grain-articles.csv           # 公众号文章核心观点提取
├── grain-x-posts.csv            # X帖子/观点记录
├── grain-tests.csv              # 工具测试记录
├── grain-projects.csv           # 项目进展记录
└── _topics/                     # 按主题整理的素材文件
    ├── ai-tools.md              # AI工具相关素材
    ├── ai-scenes.md             # AI应用场景素材
    ├── ai-trends.md             # AI行业趋势素材
    └── lessons-learned.md       # 踩坑记录和经验总结
```

## CSV 格式

### grain-articles.csv（公众号文章观点提取）

```csv
date,source,title,core_point,scene,data,attitude,reuse_as
2025-03-15,公众号,"我用Claude写了一个完整项目","Claude在处理长文本结构化任务上比GPT强","用Claude整理5000字会议录音","前三段直接可用，后面需要手动调整人称","实测下来Claude更适合这类场景","论证/案例"
2025-04-02,公众号,"三个AI写作工具横评","工具本身不重要，写作流程才重要","对比了三个工具写同一篇文章","信息密度差异明显","不要迷信工具，先搞清楚自己的写作流程","开头/观点"
```

字段说明：
- `date`：发布日期
- `source`：来源平台（公众号/X/朋友圈/测试记录）
- `title`：原文标题或主题
- `core_point`：核心观点，一句话概括
- `scene`：具体使用场景
- `data`：可引用的数据或具体细节
- `attitude`：你的判断/态度
- `reuse_as`：这个素材适合用在文章的什么位置（开头/论证/案例/结尾/对比）

## 搜索方法

### 方法A：直接搜索CSV（推荐首选）

用 Grep 工具在 CSV 文件中搜索关键词。

**搜索技巧：**
- 用 `|` 连接多个关键词（OR匹配）
- 用 `.*` 匹配中间内容
- 开启 `case_insensitive: true`
- 使用 `context: 2` 查看上下文

**示例搜索：**

搜索"Cursor相关素材"：
```
工具：Grep
模式：cursor|Cursor|agent模式
路径：_materials/
输出：content, line_number
```

搜索"写作流程相关"：
```
工具：Grep
模式：写作|流程|公众号.*方法
路径：_materials/grain-articles.csv
输出：content, line_number
context: 2
```

搜索"踩坑记录"：
```
工具：Grep
模式：踩坑|出错|失败|问题|bug
路径：_materials/grain-tests.csv
输出：content, line_number
```

### 方法B：按主题文件查找

当需要某个领域的系统性素材时，直接打开 `_topics/` 下的主题文件。

流程：
1. 根据写作主题，选择对应的主题文件
2. 浏览该主题下的所有素材条目
3. 选择最相关的素材，按改写标准扩写

## 素材改写标准

找到原始素材后，不能直接复制粘贴，需要扩写成长文素材。

### 改写规则

✅ **正确做法：**
1. **扩展上下文** — 把短帖扩写成完整段落，补充时间、场景、推理过程
2. **保留态度** — 保留真实的吐槽和明确的观点，不要美化成中性语言
3. **补充细节** — 加入具体数字、时间线、操作步骤
4. **自然过渡** — 让素材融入文章，不要生硬插入

❌ **错误做法：**
1. **直接复制** — 把短帖原封不动粘进文章
2. **过度美化** — 把吐槽改成温和的官方语言
3. **丢失个性** — 把"这个功能真难用"改成"该功能体验有待优化"
4. **编造细节** — 没有数据就不要编数字

### 改写示例

**原始素材（X帖子）：**
> 试了一下午Cursor的Agent模式，说实话比预期好，但在处理复杂泛型的时候还是会出错。

**错误改写：**
> Cursor的Agent模式在大多数场景下表现良好，但在处理复杂泛型类型时仍存在一些局限性。

**正确改写：**
> 上周花了整整一下午试Cursor的Agent模式。说实话，比我预期的好——让它重构一个3000行的TypeScript项目，大部分改得很利索。但到了复杂泛型那块就露馅了，连续三次把一个嵌套泛型改错了，最后还是我手动修的。所以结论是：日常开发够用，复杂场景别完全依赖。

## 素材库建设指南

### 初始建设（一次性）

从现有内容中提取素材，按CSV格式整理：

1. **公众号文章** → 读取已发布的文章，提取核心观点、场景、数据，填入 `grain-articles.csv`
2. **X帖子** → 导出X数据CSV，筛选有价值的帖子，填入 `grain-x-posts.csv`
3. **测试记录** → 从聊天记录、笔记中提取工具测试经历，填入 `grain-tests.csv`
4. **项目记录** → 从项目进展中提取可复用的素材，填入 `grain-projects.csv`

### 持续维护（每次发布后）

每发布一篇公众号文章或一条有价值的X帖子后，花2分钟把核心素材追加到CSV：

```
文章发布 → 提取核心观点/数据/场景 → 追加到对应CSV
```

### 主题文件维护

定期（每月一次）把CSV中的素材按主题整理到 `_topics/` 文件中，方便系统性查找。

## 使用场景

### 场景1：文章开头（真实经历切入）
搜索关键词 → 找到相关的真实经历 → 扩写成开头段落

### 场景2：观点支撑（用自己测过的来论证）
搜索关键词 → 找到测试记录 → 用具体数据支撑观点

### 场景3：案例论证（真实踩坑记录）
搜索关键词 → 找到踩坑记录 → 改写成案例段落

### 场景4：结尾收束（个人感悟升华）
搜索关键词 → 找到相关感悟 → 改写成结尾段落
