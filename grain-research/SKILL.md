# 谷雨计划 · 结构化调研

> 核心目标：调研成果实时持久化，防止会话截断丢失工作。
> 调研和写作分离——本skill只做调研，不开始写草稿。调研完成后可触发 grain-plan-writer 进入写作。

## 何时使用

- 为写文章做前期调研
- 了解某个AI工具/产品/技术的最新情况
- 搜集竞品信息或行业动态
- 任何需要多次 WebSearch 的信息搜索任务

## 执行流程

### Step 1：立即创建调研文件

**在开始搜索之前，先创建文件。** 这是核心——确保第一次搜索结果就被保存。

路径：`_knowledge_base/research-<主题>-<YYYYMMDD>.md`

初始内容：

```markdown
# [主题] 调研笔记

调研日期：YYYY-MM-DD
调研目标：[一句话说明]

## 关键问题
1. [问题1]
2. [问题2]
3. [问题3]

## 发现

（调研中逐步填充）

## 来源列表

（每次搜索后追加）
```

### Step 2：搜索并增量保存

- 每次 WebSearch 后，立即将发现追加到文件
- 每条发现附上来源 URL 和日期
- 区分一手信息（官方文档、GitHub）和二手信息（媒体评测、社区讨论）

**信息源优先级（谷雨计划标准）：**

| 优先级 | 来源类型 | 说明 |
|--------|---------|------|
| 🥇 一手 | 官方文档、GitHub repo、官方博客 | 最可靠，优先使用 |
| 🥇 一手 | 自己的测试结果 | 最有价值的素材 |
| 🥈 二手 | 权威科技媒体（The Verge、ArsTechnica等） | 需标注来源 |
| 🥈 二手 | 知名独立博主的深度评测 | 需标注来源 |
| 🥉 三手 | 社区讨论（Hacker News、Reddit） | 仅作参考，需交叉验证 |
| ❌ 忽略 | 营销软文、SEO垃圾文、洗稿文 | 不使用 |
| ❌ 忽略 | 过时信息（2024年以前的评测） | 产品可能已更新 |

### Step 3：阶段摘要

每完成 3 次搜索，在文件中保存一次阶段摘要：

```markdown
### 阶段摘要 (第N轮)

**当前关键发现：**
1. [发现1]
2. [发现2]

**待确认：**
- [还需要进一步搜索的点]

**下一步方向：**
- [接下来要搜什么]
```

### Step 4：最终简报

调研结束时，整理文件为结构化简报：

```markdown
## 调研结论

### 关键事实
1. [事实1]（来源：URL）
2. [事实2]（来源：URL）

### 来源列表
| 来源 | URL | 发布日期 | 可信度 |
|------|-----|---------|--------|
| ... | ... | ... | 高/中/低 |

### 待确认问题
- [还需要进一步验证的点]

### 写作建议
- [基于调研结果，对后续写作的建议]
  - 推荐选题角度：...
  - 需要补充的第一手体验：...
  - 可以引用的数据/案例：...
```

## 关键原则

- **先建文件再搜索**：确保第一次搜索结果就被保存
- **增量保存不等到最后**：每次搜索后立即追加
- **调研和写作分离**：本skill只做调研，不开始写草稿
- **标注可信度**：区分一手信息（官方）和二手信息（媒体/社区）
- **忽略过时信息源**：产品迭代快，2024年以前的评测参考价值有限
- **主动追问**：如果搜索结果信息不足，主动告诉用户"这块信息不够，你有没有第一手体验可以补充？"

## 调研类型

### 工具调研（最常见）
针对某个AI工具/产品，搞清楚：它能干嘛、好不好用、适合什么场景、跟同类比怎么样。

关键问题模板：
- 这个工具解决什么问题？
- 最新版本有什么功能？
- 用户反馈如何？（GitHub issues、社区讨论）
- 跟竞品相比优劣在哪？
- 免费还是付费？定价如何？

### 趋势调研
针对某个AI领域的趋势/动态，搞清楚：发生了什么、为什么重要、对普通人有什么影响。

关键问题模板：
- 最近发生了什么？
- 为什么现在火了？
- 跟之前相比有什么变化？
- 对我们（AI应用探索者）意味着什么？

### 竞品调研
针对某个赛道的多个产品，搞清楚：有哪些玩家、各自定位、优劣对比。

关键问题模板：
- 这个赛道有哪些主要玩家？
- 各自的核心卖点是什么？
- 定价策略如何？
- 用户在用哪个？为什么？

## 输出位置

- 调研笔记：`_knowledge_base/research-<主题>-<YYYYMMDD>.md`
- 长期知识：`_knowledge_base/<分类>/<主题>-<YYYYMM>.md`

## 与其他skill的关系

- 调研完成后 → 用户可触发 `grain-topic-gen` 确定写作方向
- 调研文件将作为 `grain-plan-writer` 的输入素材
- 如果调研中发现的事实需要在文章中核实 → 可由 `grain-proofreading` 交叉检查
- 如果调研中发现的信息适合长期留存 → 保存到对应的 `_knowledge_base` 分类目录
