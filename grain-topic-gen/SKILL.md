# 谷雨计划 · 选题生成

> 生成3-4个公众号文章选题方向，每个选题包含标题、大纲、优劣分析。
> 与 grain-research 配合：调研完成后触发本skill确定写作方向，再进入 grain-plan-writer 写作。

## 何时使用

- 调研完成后，确定写作方向
- 用户说"给几个选题方向"、"写什么好"
- 用户给了素材但不确定怎么切入
- 周期性选题规划（比如"这周写什么"）

## 选题质量标准

每个选题必须通过谷雨计划的实用性质检（来自 grain-plan-writer）：

1. **有没有"我试过了"的痕迹？** — 第一手体验是核心竞争力，没有亲身经历的选题标注"需要先实测"
2. **读者看完能干嘛？** — 必须有一个可执行的结论或行动建议，不能只是"涨了见识"
3. **这个选题是不是只有我们能写？** — 如果随便一个AI博主都能写出差不多的，说明没有差异化

## 选题类型

对齐 grain-plan-writer 的5种文章类型，选题从这些方向生成：

### 1. 实测报告型
- **角度**：我替你试了 + 真实数据
- **适用**：新工具、新功能、新模型
- **工作量**：⭐⭐⭐（需要真实测试）
- **核心问题**：这个东西到底好不好用？

### 2. 场景拆解型
- **角度**：一个具体场景 + 手把手拆解
- **适用**：某个AI应用场景的完整解决方案
- **工作量**：⭐⭐（需要验证步骤可复现）
- **核心问题**：这个场景普通人怎么玩？

### 3. 热点解读型
- **角度**：独立判断 + 不跟风
- **适用**：AI行业热点事件、新产品发布
- **工作量**：⭐（主要靠思考和调研）
- **核心问题**：这件事对普通人意味着什么？

### 4. 项目记录型
- **角度**：真实进展 + 踩坑记录
- **适用**：自己在做的AI项目
- **工作量**：⭐（记录真实过程即可）
- **核心问题**：我正在做这件事，进展如何？

### 5. 工具对比型
- **角度**：同类横评 + 明确结论
- **适用**：同一赛道多个工具的选择
- **工作量**：⭐⭐⭐（需要都试一遍）
- **核心问题**：到底该选哪个？

## 输出模板

每个选题按以下格式输出：

```markdown
## 选题N：[吸引人的标题]

**类型**：实测报告型 / 场景拆解型 / 热点解读型 / 项目记录型 / 工具对比型
**核心角度**：一句话说清楚这个选题的独特切入点
**工作量评估**：⭐ ~ ⭐⭐⭐
**是否需要实测**：是/否（如果需要，标注需要测什么）

### 大纲
1. 开头：[具体场景或结论]
2. 核心章节1：[标题]
3. 核心章节2：[标题]
4. 结论：[明确的可执行建议]

**预计字数**：2000-5000字

### 优势
- ✅ [优势1]
- ✅ [优势2]

### 劣势/风险
- ⚠️ [劣势1]
- ⚠️ [劣势2]

### 差异化
- 这个选题只有我们能写是因为：[说明]
```

## 标题公式

适合谷雨计划风格的标题策略：

**实测型：**
- "我试了X，结果跟想的不太一样"
- "X到底好不好用？我测了一周"
- "花了Y小时，我搞明白了X的真相"

**场景型：**
- "用X做Y，完整流程分享"
- "一个让Z省了N小时的AI用法"

**观点型：**
- "X这件事，我觉得是这样的"
- "别被忽悠了，X没那么神"

**对比型：**
- "X vs Y，我挨个试了一遍"
- "选X还是Y？看完这篇不用纠结"

**记录型：**
- "用AI做X的第N天，说说进展"
- "这个项目踩了几个坑，记录一下"

## 注意事项

- 每个选题角度要明显不同（不要4个选题都是评测同一个工具的变体）
- 优劣分析要真实，不夸大
- 工作量评估要准确——标了"需要实测"的选题就不能纯靠搜索完成
- 等待用户选择，不假设用户会选哪个
- 如果用户提供了调研文件（grain-research的输出），优先基于调研结果生成选题
- 如果用户只给了一个模糊方向，先问清楚："你有没有第一手体验？有没有已经试过的工具？"

## 与调研文件的配合

如果 `_knowledge_base/` 下有相关的调研文件，在生成选题前先读取调研结论，基于调研结果生成更有针对性的选题。避免生成需要大量额外调研的选题。
